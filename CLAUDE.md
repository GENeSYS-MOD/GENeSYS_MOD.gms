# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Project

GENeSYS-MOD (GAMS) — the original GAMS implementation of the GENeSYS-MOD
energy-system model. It reads scenario data from Excel/GDX, builds a large LP,
solves it (Gurobi / CPLEX), and writes results (gdx / csv / xls).

This is the **reference implementation**: `GENeSYS_MOD.jl` is a Julia/JuMP port
of this code. When the two diverge on equations or scenario data, **this GAMS
version is the source of truth** — align the Julia port to it, not the reverse.

## Running

- **Entry point:** `genesysmod.gms`. Run with
  `gams genesysmod.gms --switch=value ...`.
- All run options are `$setglobal` switches at the top of `genesysmod.gms`
  (lines ~25–93), each guarded by `$if not set` so the command line overrides
  the default. Key ones:
  - `model_region` (`europe` / `middleearth`) — selects `data_file` and the
    scenario-data include.
  - `emissionPathway` (`NECPEssentials` / `REPowerEU` / `Green` / `Trinity`) —
    selects the Europe input Excel and the `$ifthen` branch inside the scenario
    file.
  - `solver` (`Gurobi` default), `threads`.
  - `elmod_nthhour`, `elmod_starthour` — time-series reduction (every nth hour).
  - `switch_read_data_long` (default `1`) — chooses the data loader.
- **Inputs:** `%inputdir%` (`Inputdata\`) holds the `RegularParameters_*` and
  `Timeseries_*` Excel/GDX, generated by the `GENeSYS_MOD.data` repo.

## Architecture

`genesysmod.gms` is the driver; it `$include`s the pieces in this order:

1. `genesysmod_dec.gms` — set / parameter / variable / equation declarations.
2. `genesysmod_dataload_long.gms` (or `genesysmod_dataload.gms`) — reads input
   data; helpers in `genesysmod_dataload_functions.gms`.
3. `genesysmod_settings.gms` — run settings (modelled years, regions, ...).
4. `genesysmod_bounds.gms` — general variable bounds.
5. `genesysmod_scenariodata_%model_region%.gms` — scenario overrides + extra
   equations (`genesysmod_scenariodata_europe.gms`,
   `..._middleearth.gms`). The Europe file branches on `emissionPathway`.
6. `genesysmod_errorcheck.gms` — input sanity checks.
7. `genesysmod_equ.gms` — the full equation set (objective `cost` + all
   constraints). Largest file; defines the LP.
8. `solve genesys minimizing z using lp;`
9. `genesysmod_variable_parameter.gms` — post-solve derived quantities.
10. `genesysmod_results.gms` — result output (gdx / csv / xls);
    `genesysmod_additional_results.gms`, `genesysmod_levelizedcosts.gms`.

Other files: `genesysmod_dispatch.gms` (dispatch mode),
`genesysmod_timeseries_reduction.gms`, `genesysmod_employment.gms` (optional
module), `genesysmod_interpolation.gms`. `Untitled_*.gms` are scratch.

## Conventions & gotchas

- **Never hardcode data in the model code.** Numbers (costs, factors, caps,
  bins, prices, tech/region lists) belong in the input data pipeline (the data
  repo's CSVs -> converted Excel), never in equations, declarations, or code
  constants. Design every new feature region- and use-case-agnostic, keyed by
  data with neutral defaults when rows are absent (patterns: the
  GroupTotalAnnual*Capacity subset limits, the DispatchData_* dispatch cost
  config). If unsure how to design something universally, ask and propose
  options before implementing.

- **`$` (dollar) conditions** are GAMS conditionals: `X(i)$(cond) = ...` assigns
  only where `cond` is true; in an equation `eq(i)$(cond)..` defines the
  equation only for those `i`. In the Julia port these map to `if`/guarded
  loops — when porting, the `$` condition and the loop guard must be logically
  identical. `not X` (i.e. `X = 0`) maps to `== 0`.
- **Index order matters.** A parameter's argument order in GAMS
  (e.g. `ModalSplitByFuelAndModalType(r,f,mt,y)`) must match how the Julia side
  indexes the same container. Mismatched order is a silent data bug.
- **Scenario logic lives in two switch layers:** `model_region` picks the
  scenario file; `emissionPathway` picks the `$ifthen` branch inside it. Both
  must be reproduced when aligning the Julia scenario data.
- **`$ifthen` / `$setglobal` / `$include`** are compile-time dollar-control
  directives (resolved before execution), distinct from runtime `$` conditions.
- `genesysmod.gms` rewrites `data_file` / `hourly_data_file` based on
  `emissionPathway` and `model_region` (lines ~113–132) — the command-line
  `data_file` is overridden for the Europe/MiddleEarth presets.
- Solver options are written inline to `gurobi.opt` / `cplex.opt` via
  `$onecho ... $offecho` (Gurobi `method 2` = barrier, `barhomogeneous 1`).

## Relationship to the other repos

- `GENeSYS_MOD.jl` — the Julia/JuMP port. Mirror equation and scenario changes
  there after changing them here.
- `GENeSYS_MOD.data` — generates the input Excel consumed by both this model and
  the Julia port.
